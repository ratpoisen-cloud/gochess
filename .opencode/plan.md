# План проверки и тестирования проекта GoChess

## 🤖 Промпт для QA-Агента (Проверка работоспособности)

**Роль:** Senior QA Automation Engineer / Архитектор.

**Контекст:** Проект GoChess — это веб-приложение для игры в шахматы в реальном времени, построенное на React (Vite), Tailwind CSS, `chess.js` и Supabase (Auth, Database, Realtime). 

**Задача:** Провести комплексную проверку текущего состояния кодовой базы. Твоя цель — убедиться, что базовый функционал работает корректно, нет критических ошибок компиляции/типизации, и архитектура соответствует заявленному стеку.

**Шаги для проверки:**

1. **Анализ сборки и зависимостей:**
   - Проанализируй `package.json` и конфигурацию Vite (`vite.config.ts`, `tsconfig.json`).
   - Убедись в отсутствии конфликтующих зависимостей.
   - Проверь проект на наличие очевидных ошибок TypeScript (особенно в типах из `types/index.ts`).

2. **Проверка авторизации (Текущий фокус):**
   - Изучи файл `lib/supabase.ts` и `stores/authStore.ts`.
   - Проверь компонент `AuthModal.tsx` и хук `useAuth.ts`.
   - Убедись, что логика логина, регистрации и сохранения сессии написана безопасно и корректно обрабатывает ошибки (try/catch).

3. **Проверка шахматной логики и UI:**
   - Проанализируй компоненты `pages/LocalPage.tsx` и `components/ChessBoard.tsx`.
   - Убедись, что интеграция с `chess.js` не нарушает иммутабельность состояния (состояние всегда синхронизируется через FEN).

4. **Проверка файловой структуры и чистоты кода:**
   - Проверь импорты в основных файлах (нет ли битых путей, например, проблем с кириллицей в путях или неправильных алиасов).
   - Убедись, что логика разделена согласно правилам проекта (интерфейс отдельно, работа с БД отдельно).

**Ожидаемый результат:** 
Предоставь структурированный отчет. Если есть ошибки (особенно в логике авторизации) — укажи проблемный файл и предложи точный код для исправления. Если структура стабильна, подтверди готовность к следующему этапу разработки.
