# Проект: GoChess
**Описание:** Веб-приложение для игры в шахматы с друзьями в реальном времени.

## 🛠 Технологический стек
- **Frontend:** React (Vite)
- **UI/Стилизация:** Tailwind CSS
- **Шахматная логика:** `chess.js` (валидация ходов, состояние партии)
- **Отображение доски:** `react-chessboard` (или аналог)
- **Backend / База данных:** Supabase (PostgreSQL, Auth, Realtime)

## 📐 Архитектура и База Данных (Supabase)
Основные таблицы, с которыми мы работаем:
1. `profiles`: профили пользователей (связано с Supabase Auth).
2. `games`: активные и завершенные партии (id, white_player_id, black_player_id, status, fen_state).
3. `moves`: история ходов для конкретной партии.

*Синхронизация:* Используем Supabase Realtime для мгновенной передачи ходов и реакций (графические PNG-эмодзи).

## 🤖 Правила для ИИ-ассистента (Vibecoding Rules)

При написании кода для этого проекта строго соблюдай следующие правила:

1. **Модалки:** Все модалки настройки игры (цвет, сложность, режим) должны следовать стандарту: описание вверху, полноширинные кнопки `primary`, кнопка «Отмена» краснеет при наведении (через `hover:!bg-[var(--danger-soft)]`, текст остается белым).
2. **Графические Эмодзи:** Всегда используй ассеты из `public/emojis/board/` для реакций на доске. Константы в `src/lib/emojis.ts`.
3. **Брендинг:** Основной зеленый — `#7eb87e`, основной молочный — `#e8e8d8`. Края элементов — `var(--radius-14)`.
4. **Запросы:** Для новых игровых взаимодействий используй `RequestModal`.
5. **Фокус на чистоте:** Пиши модульный, читаемый код. Разделяй логику шахмат (движок), интерфейс (доску) и работу с сетью (Supabase).
6. **Лобби:** При отображении списка игр всегда используй `select('*')` для корректного расчета статуса хода.
7. **Шаги (Step-by-Step):** Не пытайся написать всё приложение за один ответ. Решай задачу по частям, жди подтверждения работоспособности.
8. **Безопасность (RLS):** При написании SQL-миграций для Supabase всегда учитывай Row Level Security (RLS). Пользователи должны изменять только свои партии.
9. **Обработка ошибок:** Всегда оборачивай сетевые запросы к Supabase в `try/catch` и выводи понятные сообщения об ошибках.
10. **Состояние игры:** Всегда опирайся на FEN-строку (Forsyth–Edwards Notation) как на единственный источник истины о состоянии доски.
11. **Никаких заглушек ради заглушек:** Если пишешь функцию, пиши её рабочую версию. Если нужен сторонний пакет, укажи команду для его установки.
