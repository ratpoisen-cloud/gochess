# Проект: GoChess
**Описание:** Веб-приложение для игры в шахматы с друзьями в реальном времени.

## 🛠 Технологический стек
- **Frontend:** React (Vite)
- **UI/Стилизация:** Tailwind CSS
- **Шахматная логика:** `chess.js` (валидация ходов, состояние партии)
- **Отображение доски:** `react-chessboard`
- **Backend / База данных:** Firebase (Firestore, Auth, Hosting)
- **CI/CD:** GitHub Actions (авто-деплой на Firebase Hosting)

## 📐 Архитектура и База Данных (Firebase)
Основная коллекция в Firestore:
1. `games`: активные и завершенные партии.
   - Поля: `room_code`, `white_player_id`, `black_player_id`, `pgn`, `fen`, `game_state`, `turn`, `reactions[]`, `undo_request`, `draw_request`.

*Синхронизация:* Используем Firestore Realtime (`onSnapshot`) для мгновенной передачи ходов и реакций.

## 🤖 Правила для ИИ-ассистента (Vibecoding Rules)

При написании кода для этого проекта строго соблюдай следующие правила:

1. **Модалки:** Все модалки настройки игры (цвет, сложность, режим) должны следовать стандарту: описание вверху, полноширинные кнопки `primary`, кнопка «Отмена» краснеет при наведении (через `hover:!bg-[var(--danger-soft)]`, текст остается белым).
2. **Графические Эмодзи:** Всегда используй ассеты из `public/emojis/board/` для реакций на доске. Константы в `src/lib/emojis.ts`.
3. **Брендинг:** Основной зеленый — `#7eb87e`, основной молочный — `#e8e8d8`. Края элементов — `var(--radius-14)`. Элементы UI используют шрифт `Press Start 2P`.
4. **Запросы:** Для новых игровых взаимодействий (отмена хода, ничья) используй `RequestModal`.
5. **Фокус на чистоте:** Пиши модульный, читаемый код. Разделяй логику шахмат (движок), интерфейс (доску) и работу с сетью (Firebase).
6. **Транзакции:** Для действий, требующих атомарности (вход в комнату, отправка реакции), всегда используй `runTransaction`.
7. **Безопасность:** Правила доступа хранятся в `firestore.rules`. Деплой правил осуществляется через `npx firebase deploy --only firestore`.
8. **Обработка ошибок:** Всегда оборачивай сетевые запросы в `try/catch` и выводи понятные сообщения через `addToast`.
9. **Состояние игры:** Всегда опирайся на PGN/FEN как на источник истины. В `GamePage` слушатель `onSnapshot` является главным триггером обновления UI.
10. **Ассеты:** При использовании SVG или PNG путей всегда учитывай константу `BASE` (из `import.meta.env.BASE_URL`), чтобы избежать битых ссылок при деплое.

## ✅ Ключевые достижения
- **End-Game Experience:** Реализован "взрыв" пиксельных конфетти из центра доски или клетки короля, падение проигравшего короля (90°) и контекстные эмоджи (`win`, `surrender`, `chekmate`, `draw`).
- **Lobby UX:** Добавлены фильтры по типу игры (Все, Онлайн, С ботом, Локально) и индикация "Ваш ход" / "Ход соперника" в списке партий.
- **Share Flow:** Улучшено модальное окно создания комнаты, добавлены кнопки быстрого шеринга в Telegram и VK, а также блок приглашения друга внутри игровой комнаты.
- **Стабильность:** Исправлен баг "цифры ноль" при загрузке онлайн-доски и ошибки типизации `Move`.
